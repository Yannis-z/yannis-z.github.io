解析移动端像素尺寸

内容概览

  • 像素的概念
  • 分辨率的概念
  • 设计与开发中的尺寸
1、像素的概念
  • 逻辑像素 [pt/dp] ——视觉上固定的物理长度。iOS系统中逻辑像素单位是pt,Android系统中是dp。
  • 物理像素 [px] ——设备硬件可以达到的像素数量，和渲染的精细度有关。
  • 渲染像素 [px] ——软件可以达到的像素数量，有时比物理像素高。
2、分辨率的概念
  • 逻辑分辨率 [pt/dp] ——视觉的固定宽度*高度
  • 物理分辨率 [px] ——横像素数*竖像素数（硬件可达到的像素数量）
  • 渲染分辨率（显示分辨率）[px] ——横像素数*竖像素数（软件可达到的像素数量）
  • PPI（图像分辨率、屏幕像素密度）——Pixel Per Inch
  • DPI（常用于表示印刷精度）——Dots Per Inch
3、设计与开发中的尺寸
3.1 IOS
设计
  • 设计采用单位为px，设计尺寸参考渲染分辨率
  • 画布设置——画布的尺寸设置参照渲染分辨率（大部分情况下，渲染分辨率和物理分辨率是一致的） 有些机型可以调整渲染分辨率）。
    在Photoshop中我们采用的图像分辨率是72dpi(PS中DPI=PPI), 已知1pt=1/72Inch, ppi=px/1Inch,所以当ppi=72时，画布中1px=1pt。
  • 特定元素尺寸——设计中我们需要参考ios规范来确定一些特定元素的尺寸，比如状态栏高度，导航栏高度，标签栏高度，系统图标大小等等，来保证一些元素在不同尺寸的设备中都有一致的大小（因为不同尺寸设备的PPI相同时，一个像素的实际大小也是相同的）。
开发
  • 开发常用单位为PT,字体单位为PX。开发尺寸参照逻辑分辨率。
  • 设计稿的应用——因为渲染分辨率（设计尺寸）和逻辑分辨率（开发尺寸）成比例（2或3倍），设备中1pt就会有2个或者3个像素去渲染。又因为ps中的dpi=ppi=72,1pt=1/72Inch,所以设计稿中1pt=1px，开发时我们需要将设计稿中的元素尺寸除以相应的倍率进行应用（2或3）,某些特定元素的尺寸在除以倍率后视觉上的显示大小是相同的。
3.2 Android
设计
  • 设计采用单位是PX。
  • 画布设置——由于安卓手机屏幕碎片化严重，它把各种设备的屏幕像素密度（PPI）划成了多个范围区间，给不同范围的设备定义了不同的倍率，来保证显示效果相近。所以采用了几种主流的PPI和对应的屏幕像素，在不同的PPI下对一些图标有固定的尺寸规范。
开发
  • 开发常用单位为DP(DIP),字体单位为SP。
  • 1DP定义为屏幕像素密度为160PPI时的1px。
    在MDPI时，1PD=1PX，所以1DP的尺寸是固定的。开发中Android系统对根据几种典型的屏幕像素密度进行适配。

Docker

Dillinger is very easy to install and deploy in a Docker container.

By default, the Docker will expose port 8080, so change this within the Dockerfile if necessary. When ready, simply use the Dockerfile to build the image.

This will create the dillinger image and pull in the necessary dependencies. Be sure to swap out ${package.json.version} with the actual version of Dillinger.

Once done, run the Docker image and map the port to whatever you wish on your host. In this example, we simply map port 8000 of the host to port 8080 of the Docker (or whatever port was exposed in the Dockerfile):

Verify the deployment by navigating to your server address in your preferred browser.
